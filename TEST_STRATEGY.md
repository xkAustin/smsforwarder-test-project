# SmsForwarder测试策略

## 测试背景：

## 测试目标：

- 验证核心功能在不同配置与规则下的正确性
- 保证消息转发接口（Webhook / HTTP）的字段完整性与契约一致性
- 验证系统在高频消息场景下的稳定性与性能边界
- 构建可自动执行的测试体系，支持持续集成与回归测试
- 为后续白盒测试与代码重构提供安全保障

## 测试对象：SmsForawrder（<https://github.com/pppscn/SmsForwarder）>

## 测试范围：

### 黑盒测试：从用户视角对整个系统进行完整的黑盒测试

- 短信/来电/通知的监听和触发
- 不同转发通道的配置正确性
- 不同转发规则的配置正确性
- 转发失败时的错误提示
- 权限能否正常获取和访问
- 后台保活
- 权限异常、网络异常等场景的异常提示
- 错误日志是否有内容

### 接口测试：以消息转发接口为核心，验证系统对外输出的正确性：

- Webhook 请求方法、路径、Headers、Body 校验
- 字段完整性、数据格式、编码方式验证
- 时间戳、签名、幂等性（如有）
- 转发失败时的重试行为（次数、间隔）
- 接口异常（4xx / 5xx / 超时）处理能力

### 自动化测试：使用Python自动化测试框架实现可回归测试

- 使用 pytest 组织测试用例
- 使用 mock server 接收并验证 Webhook 请求
- 支持一键运行全部接口与核心功能测试
- 为 CI/CD 集成提供基础能力

### 性能测试：验证系统在高负载场景下的稳定性和性能表现

- 高频短信/通知触发场景
- 连续转发情况下的响应时间
- 转发失败重试对系统资源的影响
- 长时间运行下的稳定性观察

### 白盒测试：源码测试

- 核心规则匹配逻辑的单元测试
- 通道适配器与数据处理模块的单元测试
- 边界条件与异常分支覆盖

### 非测试范围：
适配机型, 操作系统级别bug, 第三方平台bug

## 测试环境：
测试设备：AVD Medium Phone
系统：Android 16.0
自动化环境：Python 3.14.2, pytest 9.0.2, requests 2.32.5, mock 5.2.0
